# Echo Print Check

This script processes `.xml` files generated by the Echo 550 liquid handler, searching for skipped wells where transfers were unsuccessful. It generates `.csv` files listing only the missing wells, which can be used to re-run the process and transfer the required volumes.

## Features

- Reads Echo 550 `.xml` files containing print results.
- Identifies skipped wells for a given plate barcode (source or destination).
- Groups files by transfer time (within 30 minutes).
- Outputs `.csv` files with missing wells for re-processing.

## Requirements

- Python 3.6+
- Standard libraries: `os`, `sys`, `operator`, `csv`, `datetime`, `xml.dom.minidom`, `xml.etree.ElementTree`

## Usage

1. Place your Echo `.xml` files in the sample directory:
   - `/Users/flavia/PycharmProjects/Echo_survey_check/sample`

2. Run the script from the command line:
    ```bash 
   - `<source_or_destination>`: `0` for source plate, `1` for destination plate
   - `<barcode>`: Plate barcode to search
    - `<output_directory>`: Directory to save the output `.csv` files
     ```bash
     python echo_survey_check.py <source_or_destination> <barcode> <output_directory>
     ```
3. Output `.csv` files will be created in:
   - `/Users/flavia/PycharmProjects/Echo_survey_check/output`

## Output

Each `.csv` file contains:
- File name
- Source barcode
- Source well
- Destination barcode
- Destination well
- Actual volume transferred
- Volume transferred

These files can be used to re-run the Echo process for only the missing wells.

## File Structure

- `main.py`: Main script
- `/sample`: Directory for input `.xml` files
- `/output`: Directory for generated `.csv` files
